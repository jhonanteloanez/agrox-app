## 🛠 Project Overview
**AgroX** is an agricultural management system designed for individual farmers (Plan P1) and cooperatives (Plan P2). It includes property management, crop tracking, and inventory control.

## 🏗 Tech Stack
- **Frontend**: React + Vite + TypeScript + Tailwind CSS + Lucide React.
- **Backend**: Node.js + Express + TypeScript + Prisma.
- **Database**: PostgreSQL (Supabase) + custom PL/pgSQL functions (`fn_*`).
- **Auth**: Custom JWT-based authentication.

## 📍 Current State (March 14, 2026)
### 1. Core Modules Implemented:
- **Authentication**: Login/Register, JWT flow, and Role/Plan detection.
- **Properties & Plots**: Creation, list, and detailed view of agricultural land and its subdivisions.
- **Crops**: Phenological tracking of crops linked to plots.
- **Inventory**: Stock management for inputs (seeds, fertilizers, etc.) with real-time alerts.

### 2. Recent Architectural Changes:
- **Global Layout**: Implemented a `Sidebar` and `Layout` component in the frontend to provide persistent navigation across all modules.
- **Plan Logic**: Corrected the logic to fetch `plan_code` from the `subscription` table (joined with `org_user_role`) instead of `organization.plan_type`.

## ⚙️ Key Technical Details
- **BigInt Handling**: The project uses `serializeRow` in the backend to convert BigInt values (from PostgreSQL) to strings before sending them to the frontend to avoid JSON errors.
- **Database Functions**: Most logic resides in Supabase PL/pgSQL functions (`fn_inventory_list`, `fn_create_crop`, etc.).
- **Inventory Logic**: Stock exits (`SALIDA`, `CONSUMO_CULTIVO`) are handled by converting the quantity to negative before calling the DB function.
